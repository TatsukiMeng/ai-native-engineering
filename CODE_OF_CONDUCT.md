# 项目行为准则

这个仓库欢迎不同背景、不同经验层级的贡献者，也欢迎人类作者与 AI 智能体协作参与。

但无论是写书、提 Issue、发 PR、评论章节，还是讨论工具、模型和工程路线，我们都希望这里保持一种很明确的氛围：

- 尊重人，也尊重事实。
- 可以有观点，但不要对人下手。
- 可以争论路线，但不要把讨论变成情绪宣泄。
- 欢迎 AI 协作，但不要拿 AI 当挡箭牌。

这份行为准则适用于仓库内外所有与本项目相关的公共协作空间，包括但不限于：

- GitHub Issues
- Pull Requests
- Discussions
- Commit、评审与代码评论
- 文档内容评论、补充与勘误讨论

## 我们鼓励什么

我们鼓励下面这些行为：

- 对问题本身保持尖锐，对人保持基本尊重。
- 在提出判断时尽量给出上下文、证据、复现步骤或资料来源。
- 对新手友好，不把“你应该早就知道”当成沟通方式。
- 承认不确定性，愿意修正自己的结论。
- 在引用外部资料、模型输出或 AI 生成内容时，尽量说明来源和边界。
- 针对文稿、代码、结构提出具体可执行的建议，而不是只丢结论。
- 对不同技术路线保持开放，但不把“站队”当贡献。

## 我们不接受什么

下面这些行为不被接受：

- 人身攻击、羞辱、嘲讽、贴标签、阴阳怪气式持续挑衅。
- 基于性别、年龄、地区、职业背景、学校、经验水平等进行歧视或排斥。
- 恶意引战、刷屏、跑题灌水、故意制造对立。
- 未经允许公开他人隐私、私人信息、工作信息或聊天记录。
- 明知内容未经核验，仍大段提交 AI 幻觉、伪造引用、伪造经历或伪造实验结论。
- 把“这是 AI 生成的”当作不对内容负责的理由。
- 对维护者、贡献者或读者进行持续骚扰，包括在仓库外延续针对性攻击。

## 关于 AI 协作的额外约定

这是一个明确鼓励 AI 协作的项目，但也正因为如此，我们会额外看重两件事：

第一，**AI 不是免责理由**。

无论内容是人写的，还是 Cursor、Claude、GPT、Codex 或其他工具辅助生成的，最终提交者都需要对结果负责。

第二，**生成速度不能高于事实质量**。

如果一段内容的来源、引用、实验过程、结论边界都说不清，那它就不该因为“写得快”被直接合入。

## 维护者的处理方式

如果出现违反本准则的行为，维护者可以根据情况采取下面这些措施：

- 公开提醒或私下提醒
- 要求修改措辞、补充来源或收回不当内容
- 删除评论、关闭讨论、锁定线程
- 拒绝或关闭相关 PR / Issue
- 在严重或持续违规时，限制后续参与

处理时会优先看三件事：

- 行为本身是否越界
- 是否属于一次性失误，还是持续性模式
- 对项目协作氛围和事实质量造成了多大影响

## 如何反馈

如果你遇到以下情况：

- 讨论已经越过“对事不对人”的边界
- 有人持续骚扰、羞辱或恶意攻击
- 有内容存在明显伪造、误导或恶意操纵

可以通过下面两种方式反馈：

- 一般问题：直接在对应 Issue / PR / Discussion 中指出
- 敏感问题：通过 GitHub 私信联系仓库维护者 `@TatsukiMeng`

## 最后一句话

这个项目欢迎锋利、具体、能落地的讨论。

但我们希望这里的锋利，始终是冲着问题，不是冲着人。
