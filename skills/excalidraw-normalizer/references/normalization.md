# 规范化检查清单

## 按这个顺序执行

1. **先选布局模型**
   - star：一个中心，多个叶子节点
   - ring / polygon：peer-to-peer 或去中心化拓扑
   - lane：角色或工作流对照
   - grid：分组组件
   - hierarchy：父子层级系统

2. **规范化关系**
   - 用 `containerId` 绑定节点文字
   - 用 `startBinding` 和 `endBinding` 绑定连接
   - 给被连接图形补齐匹配的 `boundElements`

3. **规范化几何**
   - 用可重复的模式重算节点位置，而不是临场乱拖
   - 同一视觉层级内保持一致的节点尺寸
   - 除非不对称本身有语义，否则尽量保持间距和角度对称

4. **规范化连线策略**
   - 简单拓扑优先直连
   - 只有在需要避免交叉或边缘歧义时，才用 orthogonal 或多段连接
   - 如果连线质量依赖编辑器路由，就把手写 `points` 视为临时结果

5. **规范化导出样式**
   - 默认透明背景
   - 节点默认 `solid` 填充
   - 连线颜色优先兼容夜间模式
   - 透明导出时，线条比默认稍粗

## 停止条件

满足下面几点再停止：

- 拖动节点时，标签和连接都跟着走
- 这张图的布局可以用一句几何描述说清楚
- 没有哪根连接线看起来像是从某个随机像素点射出来的
- 透明 PNG 在深色背景下仍然可读

## 不要接受这些失败状态

- 只是“碰巧看起来在中间”的浮动标签
- 视觉上碰到节点、但实际上没有绑定的自由线
- 只要拖一次就会坏掉的“差不多锚点”
- 除非白底本身属于图语义，否则不要导出白色实底背景
