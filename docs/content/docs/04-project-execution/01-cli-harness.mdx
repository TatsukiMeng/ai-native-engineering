# 这一章要解决什么问题

很多高价值任务，最后都不发生在聊天框里，而是发生在终端里：安装依赖、跑构建、执行脚本、拉起环境、验证产物、处理失败。

如果 AI 只能“建议”，不能进入受控执行闭环，那么它在项目后半段的价值会很快见顶。

# 这一章会怎么展开

- 先讲为什么 CLI 仍然是高价值任务的真实施工现场
- 再讲 Harness 到底补上了哪些关键能力：权限、执行、回显、验证、失败收敛
- 最后讲怎么把“会调用 Bash”升级成“能在真实工程里稳定跑流程”

:::important[占位说明]
这篇后续会补入 Codex CLI / Claude Code 一类 agentic CLI 的工作方式、命令执行边界、验证链路和常见失败模式。
:::

# 读完这一章，读者应该带走什么

- 终端能力不是附属品，而是 AI 工程闭环的重要一环
- Harness 的核心不是炫酷，而是可控执行与机械验证
- 没有受控执行环境，很多高价值任务根本无法稳定落地
