# 这一章要解决什么问题

一旦项目不再只跑在浏览器里，很多隐藏复杂度就会突然冒出来：宿主通信、本地文件能力、IPC、打包差异、平台兼容、微前端边界。

这些问题如果没有先定义清楚，AI 很容易生成一堆“看起来能连起来，实际上根本落不了地”的代码。

# 这一章会怎么展开

- 先讲跨端与宿主问题为什么本质上是边界问题，不是 UI 问题
- 再讲 Tauri、Wujie 这类场景里应该先约束哪些接口与通信协议
- 最后讲控制面在跨端阶段要如何升级，才能避免平台差异把协作拖垮

:::note[占位说明]
这篇后续会补入宿主通信边界、跨端能力清单、平台差异风险点与常见约束模板。
:::

# 读完这一章，读者应该带走什么

- 跨端不是为了炫技，而是项目真实边界的一部分
- 平台能力越复杂，越要先定义协议和约束
- 控制面如果不升级，AI 在跨端阶段很容易从高效变成高危
