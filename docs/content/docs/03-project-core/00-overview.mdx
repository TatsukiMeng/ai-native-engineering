# 这一部分开始真正做项目

前两部分做的事情，本质上是在校正认知、补工程底线。

到了 Part 3，项目才算真正起盘：不再只是讨论“AI 应该怎么用”，而是开始回答“一个真实项目到底该怎么被稳定推进”。

这部分最重要的原则很简单：**先把主干立住，再让 AI 去加速，而不是一上来就让它四处开写。**

# 本部分的阅读地图

- `第 1 章` 先收敛需求，确定项目到底做什么、不做什么
- `第 2 章` 再把架构、数据、接口和模块边界立住
- `第 3 章` 进入后端主线，让 PostgreSQL、服务层和接口层一起长出来
- `第 4 章` 进入前端主线，把组件协议、状态边界和页面协作策略讲清楚

:::tip[这一部分不要追求“一次生成到位”]
项目真正进入实现阶段后，AI 最怕的不是任务难，而是边界没定清楚就被要求直接输出“大成品”。

所以这部分会反复强调：需求先收敛、协议先定义、主干先闭环。
:::

# 读这一部分时，要重点看什么

- 哪些东西必须先定义，不能靠生成时临场发挥
- 哪些实现链路适合先做主干，再逐步扩展
- 人类在这个阶段最该盯的，是规则、边界和验收口径
