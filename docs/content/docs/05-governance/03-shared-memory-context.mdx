# 这一章要解决什么问题

当任务开始跨天、跨人、跨 Agent 持续推进时，系统最容易出问题的地方之一，就是“记忆”本身。

该留下来的没留下，不该继续影响判断的旧信息却一直在，最后 AI 被旧实现、旧结论、旧路径反复带偏。

# 这一章会怎么展开

- 先讲共享记忆、短期 Context、决策日志、知识库各自该存什么
- 再讲顺序、频率、重复引用、时效性和迁移期新旧并存会怎样影响判断
- 最后讲怎么把 Context 从一次性输入，升级成可治理的系统资产

:::note[占位说明]
这篇后续会补入共享记忆分层策略、决策日志样例、索引方案和旧信息清理机制。
:::

# 读完这一章，读者应该带走什么

- 系统不是记得越多越好，而是要记得对、忘得掉
- Context 治理本质上是信息治理，不只是提示词技巧
- 多 Agent 场景下，记忆边界不清会直接放大混乱
