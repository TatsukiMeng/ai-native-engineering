# 为什么这里必须把项目亮出来

如果一直只讲范式、工具和控制面，很多判断都会停在“听起来对”的层面。只有把一条真实主线摆上桌面，后面的 MCP、Harness、Playwright、Rules、Skills、治理和 Swarm 才不会悬空。

# 这条主线为什么会收束成 `Our100`

我需要的不是一个好讲的 demo，而是一场真仗：它要同时带着多端、实时、隐私、自动化和长期治理压力。顺着这个要求往下推，主线项目最后收束成了 `Our100`。它不是一个轻量打卡工具，而是一套会自然逼出连接、感知、主动关怀、权限边界和平台治理问题的真实战场。
