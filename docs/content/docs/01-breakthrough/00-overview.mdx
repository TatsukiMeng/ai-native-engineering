# 这一部分到底要掰正什么

Part 1 不负责把所有概念一次讲透，它只负责先把读者脑子掰过来。

如果上来就把 AI 当成“更强一点的 Copilot”，后面几乎每一步都会走偏：任务会越做越散，Context 会越喂越乱，工具会越换越焦虑，最后把所有问题都甩给模型“还不够聪明”。

这一部分真正要完成的，是先建立一个最小但够用的心智框架：**问题很多时候不在模型，而在工作流、控制面和执行方式。**

# 本部分的阅读地图

- `第 1 章` 先解释为什么很多人会觉得 AI 很笨，本质上是工作流还停在补全时代
- `第 2 章` 再解释工具为什么不是信仰，真正该看的其实是产品形态、岗位分工和成本边界
- `第 3 章` 把 Prompt、Context、Agent Harness 串起来，让控制面的轮廓先出现
- `第 4 章` 直接把全书主线项目亮出来，让读者知道后面到底要打什么仗

:::important[这一部分的任务]
这一部分不是工具图鉴，也不是术语百科。

它的任务只有一个：让读者意识到，AI Native 开发的关键不是“哪个模型最强”，而是“怎么把能力组织成一套能稳定推进项目的工程系统”。
:::

# 读这一部分时，先带着这几个问题

- 为什么同样的模型，换个工作流体验会差这么多？
- 为什么工具越多，反而越需要先看控制面？
- 为什么这本书不拿一个小 demo，而要拿一个真实项目开刀？
