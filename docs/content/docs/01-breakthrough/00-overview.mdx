# 这一部分真正要破的，不是“不会用工具”，而是旧脑回路

很多人第一次接触 AI 编程，卡住的地方其实不是不会装工具，也不是不知道哪个模型更强，而是脑子里默认还在沿用旧开发范式：把 AI 当高级自动补全，把 Prompt 当万能钥匙，把工具选择当单选题，把所有问题都理解成“模型还不够聪明”。

Part 1 的任务，就是先把这层旧脑回路掰过来。

# 为什么这一部分要分成 5 篇文章

这 5 篇文章不是工具图鉴，也不是术语词典，而是在替后面的项目主线铺路。

- 第 1 篇先解释为什么范式已经变了
- 第 2 篇再解释为什么工具不能只看强弱
- 第 3 篇把 Prompt、Context、Harness 收束成控制面
- 第 4 篇把产品判断这层前置能力点出来
- 第 5 篇再把主线项目亮出来，让后文真正落地

如果没有这 5 篇，后面的项目推进很容易又被读回“AI 帮我写代码”的老路。
