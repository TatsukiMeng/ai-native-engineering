# 如何贡献 (Contributing)

感谢你对《AI 原生工程》项目的关注！这是一个高度自动化、AI 驱动的开源书籍项目。如果你想参与贡献（无论是修正错别字、改进代码案例还是增加新章节），请遵循以下规范。

## 🤖 AI 协作优先

本项目鼓励使用 AI 智能体（如 Cursor, Claude, GPT 等）辅助创作。在提交内容时，请优先参考项目根目录下的 [AGENTS.md](AGENTS.md)，其中详细定义了：
- **双轨协作模式**：如何在内容写作与技术发掘之间切换。
- **实战驱动 (Show, Don't Just Tell)**：代码示例的编写标准。
- **目录结构隔离**：文档与实战案例的分离规则。

## 📝 提交内容规范

1. **语言风格**：保持专业、直切要害、具备极客感。
2. **术语使用**：保持原文术语（如 Prompt, Context, Vibe Coding, Swarm），严禁生硬翻译。
3. **Fumadocs 适配**：如果你修改了 `docs/` 内容，请确保使用了 Fumadocs 原生的 MDX 组件来优化排版。

## ⚙️ Git Commit 规范

所有的 Commit 提交 **必须使用中文**，并遵循业界通用的前缀规范：

- `docs:` 修改或增加文档内容
- `feat:` 增加新的实战案例或代码功能
- `fix:` 修复内容错误、错别字、代码 Bug 或排版问题
- `style:` 调整格式（空格、缩进等），不影响逻辑
- `chore:` 辅助性质的修改（构建配置、依赖更新等）

示例：`docs: 完善第二章中关于“自洽陷阱”的论述`

## 🚀 贡献流程

1. Fork 本仓库。
2. 创建你的特性分支 (`git checkout -b feat/my-new-chapter`)。
3. 提交你的修改并保持 Commit 信息整洁。
4. Push 到分支 (`git push origin feat/my-new-chapter`)。
5. 发起 Pull Request。
